# Reverse Engineering Report: zFont 3 & Samsung Font Exploit

## 1. Internal Mechanics of zFont 3
Through decompilation and asset analysis of `com.htetznaing.zfont2.apk`, we have mapped out how zFont 3 bypasses Samsung's font signature restrictions.

### 1.1 The Template System
zFont does not "patch" the system; it generates a specially crafted APK that spoofs an official Samsung font.
- **Location:** `assets/source/samsung/`
- **Structure:** Contains zlib-compressed APK templates for various Android versions (K=KitKat, L=Lollipop, M=Marshmallow, N=Nougat, O=Oreo, P=Pie/One UI).
- **The Core Template:** The `oneui/sans` asset is a compressed APK with the package name `com.monotype.android.font.samsungsans`.

### 1.2 The Generation Process
1.  **Decompression:** The app extracts and decompresses the `sans` template.
2.  **Injection:** It overwrites `assets/fonts/Samsungsans.ttf` with the user's chosen font.
3.  **Naming:** It updates the following to match the new font's name:
    - `assets/xml/Samsungsans.xml` (the `displayname` attribute)
    - `res/values/strings.xml` (the `app_name` string)
4.  **Rebuild:** It uses an internal version of `apktool` logic to rebuild the APK.
5.  **Signing:** It signs the APK with a generic debug key.

## 2. The "Backup & Restore" Exploit (Optional on some Wear OS versions)
This was traditionally the critical part of the process. Samsung's system only allows fonts signed by Monotype or Samsung to be applied.

**CRITICAL UPDATE:** Empirical testing on modern Wear OS (Galaxy Watch 5 Pro) has shown that the "Restore" trick may be **completely optional**. In many cases, simply installing the spoofed APK generated by our patcher allows the font to appear immediately in the system settings.

### 2.1 The Traditional Cloud Method
1.  **Placeholder Phase:** The user installs the *real* Samsung Sans (signed and authorized).
2.  **Backup Phase:** The user backs up "Settings" to Samsung Cloud.
3.  **Swap Phase:** The user uninstalls the real Samsung Sans and installs the *fake* one.
4.  **Restore Phase:** The user restores "Settings" from the cloud.

### 2.2 The "Minimalist Install" Method (Verified Working on Wear OS)
On modern Wear OS (Galaxy Watch 5 Pro), the installation process is significantly more streamlined than on mobile devices:

1.  **No Placeholder Needed:** You do **not** need to install the original Samsung Sans font first.
2.  **No Restore Needed:** The Cloud Backup/Restore exploit is **not required**.
3.  **Install:** Simply sideload the APK generated by our patcher via ADB.
4.  **Apply:** Go to **Settings > Display > Font style**.
5.  **Naming Behavior:** 
    - The font will appear in the selection list with the **Custom Display Name** you provided to the patcher.
    - After selection, it may still be referred to as "Samsung Sans" in some sub-menus (due to the spoofed package name), but the visual font rendering and primary label will reflect your custom font.

## 3. PC-Based Patcher (Our Implementation)
We have created a standalone Python script that performs the same APK generation as zFont, allowing you to create your own font packages on a PC.

### 3.1 Tools Used
- **Python 3:** Orchestration.
- **7-Zip:** Extraction from zFont XAPK.
- **Apktool (v3.0.2):** Decoding and Rebuilding.
- **Uber APK Signer (v1.3.0):** Zipalign and Signing.

### 3.2 How to Use the Patcher
1.  **Location:** `apps/patcher/samsung_font_patcher.py`
2.  **Command:**
    ```bash
    python3 apps/patcher/samsung_font_patcher.py <your_font.ttf> <output_name.apk> "Your Font Name"
    ```
3.  **Output:** A signed APK that is ready for the Samsung Cloud exploit.

## 4. Usage Instructions for Wear OS (Galaxy Watch)
Since you are in a Codespace, you cannot use ADB directly. Follow these steps:

1.  **Generate the APK** using the command above.
2.  **Download the APK** from the Codespace to your local PC.
3.  **Connect to Watch via ADB** on your local machine:
    ```bash
    adb connect [Watch_IP]
    adb install output_name.apk
    ```
4.  **Perform the Exploit:**
    - Install the *real* Samsung Sans first (if not present).
    - Set font to Samsung Sans.
    - Backup Watch Settings via the Galaxy Wearable app on your phone.
    - Install your *custom* font APK (it will overwrite/replace).
    - Restore Watch Settings via the Galaxy Wearable app.

## 5. Technical Constraints
- **One UI Version:** This method works on One UI 4, 5, and 6. Some very recent security patches (One UI 6.1+) have started blocking this specific restore exploit.
- **Font Format:** Strictly use `.ttf`. The system often fails to render `.otf` fonts correctly in this spoofed package.
