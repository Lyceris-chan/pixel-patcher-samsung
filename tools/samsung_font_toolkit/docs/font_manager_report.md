# Decompilation Report: fmwos (Font Manager Wear OS)

## 1. Executive Summary
The `fmwos.xapk` is a production-grade Flutter application designed to manage and install custom fonts on Samsung Galaxy Watches (Watch 4, 5, 6, 7, and 8). The system operates on a Phone-Watch bridge architecture where the phone app acts as a resource repository and transfer conduit, while a corresponding watch-side application (likely `com.monotype.android.font.fontmanager` or a generic "File Transfer" app) handles the privileged installation of font APKs on Wear OS.

## 2. Decompilation Analysis

### 2.1 Toolchain Used
The analysis was performed using the following suite of decompilation tools to ensure maximum coverage and code readability:
*   **Jadx (v1.5.0):** Primary tool for resource decoding and high-level Java source reconstruction.
*   **Enjarify:** Used to translate Dalvik bytecode (DEX) to Java bytecode (JAR) to facilitate use with specialized Java decompilers.
*   **CFR (v0.152):** Utilized to decompile the JAR generated by Enjarify, providing superior handling of modern Java features and control flow.
*   **Vineflower (v1.10.1):** A modern fork of Fernflower (Vernflower) used to verify complex class structures and inner class relationships.

### 2.2 Architectural Overview
The application is a **Flutter-based hybrid**.
*   **UI Layer (Flutter):** Handles the font library, previews, and user interactions.
*   **Platform Layer (Java/Android):** Manages the bridge between Flutter and the Wear OS system services.
*   **Communication Layer (Wearable Data Layer):** Utilizes Google Play Services `ChannelClient` and `MessageClient` for bidirectional communication between the phone and watch.

### 2.3 Communication Protocol
The app implements a custom protocol for file transfer:
1.  **Command Structure:** Uses a JSON-based message format (`DLMessage`) with fields for `cmd`, `event`, and `result`.
2.  **Channel Pathing:** File transfers are initiated via binary channels with a structured path:
    `filetransfer<fileIndex>###<fileCount>###<totalSize>###<fileName>`
3.  **Data Flow:**
    *   Flutter triggers `sendFilesToWatch` via a MethodChannel.
    *   `MainActivity` initializes `AppDLService` (extending `PlatformDLService`).
    *   `PlatformDLService` opens a `ChannelClient` to the watch and streams the file data in 1KB chunks.

## 3. Font Installation Mechanism
Samsung Galaxy Watches (running One UI Watch based on Wear OS) do not support direct `.ttf` installation via the file system. Instead, they require fonts to be packaged as **FlipFont APKs**.

### 3.1 APK Requirements
A compatible font APK must have:
*   **Package Name:** Usually follows `com.monotype.android.font.<fontname>`.
*   **Assets:**
    *   `assets/fonts/<fontname>.ttf`: The actual TrueType font file.
    *   `assets/xml/<fontname>.xml`: A configuration file mapping the font to system droid names (Sans, Bold, etc.).
*   **Manifest:** Must include `<uses-feature android:name="android.hardware.type.watch" />` and the metadata `com.google.android.wearable.standalone`.

## 4. Manual Usage Guide (Without the App)
To install your own custom fonts without using the Font Manager app, follow these steps:

### Step 1: Prepare the Font APK
You can use a tool like **zFont 3** on your phone to generate a Samsung-compatible APK from your `.ttf` file.
1.  Select your `.ttf` in zFont.
2.  Choose "Samsung (OneUI)" as the target.
3.  Install the generated APK on your phone (it acts as a temporary container).
4.  Use an APK extractor (e.g., APK Analyzer) to save the generated APK (e.g., `com.monotype.android.font.custom.apk`).

### Step 2: Sideload to Watch via ADB
1.  **Enable Developer Options on Watch:** Go to Settings > Software Info > Tap "Software Version" 7 times.
2.  **Enable ADB & Wireless Debugging:** Go to Settings > Developer Options. Enable "ADB Debugging" and "Wireless Debugging".
3.  **Connect ADB:**
    ```bash
    adb connect [Watch_IP]:[Port]
    ```
4.  **Install the APK:**
    ```bash
    adb install com.monotype.android.font.custom.apk
    ```

### Step 3: Apply the Font
1.  On the watch, go to **Settings > Display > Font style**.
2.  Your custom font will now appear in the list. Select it to apply.

## 5. Technical Constraints
*   **Format:** Only **.ttf** files are reliably supported. **.otf** files often fail to render in the system UI.
*   **Signature Check:** Modern One UI Watch versions enforce signature checks. If a direct `adb install` fails with a signature error, you may need to use a "Backup & Restore" method (common in tools like #mono_) to bypass the restriction.
